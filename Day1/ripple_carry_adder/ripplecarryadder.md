4-Bit Ripple Carry Adder Simulation & Verification
--------------------------------------------------
The functionality of the 4-Bit Ripple Carry Adder was verified via behavioral simulation using AMD Xilinx Vivado 2022.2. The waveform below demonstrates standard binary addition across multiple test cases, handling regular additions, intermediate carries, and maximum values.

 4-Bit Ripple Carry Adder (RCA)1
 -------------------------------
 Working MechanismThe Ripple Carry Adder is a structural digital circuit built by cascading four 1-bit Full Adders in series.Carry Propagation: Each stage computes a local sum and a carry-out bit. The carry-out of bit $i$ ($C_i$) acts directly as the carry-in ($C_{in}$) for bit $i+1$.The "Ripple" Effect: Because the most significant bits depend on the carry bits generated by the lower-order bits, the carry signal must "ripple" through the entire chain from the Least Significant Bit (LSB) to the Most Significant Bit (MSB).
 Technical SpecificationsWord Length: 4 bits (A[3:0], B[3:0])Input Range: $0000_2$ to $1111_2$ ($0$ to $15$ unsigned decimal)Maximum Output Capacity: $31_{10}$ ($11111_2 \rightarrow$ Cout = 1, S = 4'hf)Critical Delay Path:$$\tau_{total} = \tau_{setup} + (N \times \tau_{carry}) + \tau_{sum}$$Where $N = 4$ stages.
 
 4-Bit BCD Adder1
 ---------------
 
 Working MechanismA BCD adder relies on a regular 4-bit binary adder but restricts valid inputs and outputs to decimal digits $0$ through $9$ ($0000_2$ to $1001_2$). It operates in a two-stage correction pipeline:
 Step 1: Binary Addition The module uses an underlying 4-bit adder block to sum $A + B + C_{in}$, yielding an intermediate binary sum ($S_{int}$) and an intermediate carry ($K$).
 Step 2: Overflow Detection If the intermediate sum is greater than $9$ ($S_{int} > 1001_2$) or if a carry occurred ($K = 1$), it means the result has drifted into the invalid pseudo-tetrad zone ($1010_2$ to $1111_2$). This condition is tracked using the Boolean combinational logic expression:$$C_{out} = K + (S_3 \cdot S_2) + (S_3 \cdot S_1)$$
 Step 3: Binary-to-Decimal Correction If $C_{out} = 1$, the logic adds a digital offset of $+6$ ($0110_2$) to the intermediate result. This skips the 6 invalid binary states, forcing the lower 4 bits to roll back into a valid 0–9 range while asserting the final decimal carry-out.
